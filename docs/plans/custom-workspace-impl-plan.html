sk/workspaces • 2026-03-05

Custom Workspace
Implementation Plan

Evolve TenantWorkspace in-place — OneToOne → M2M tenants, add a workspace_type discriminator, unified WorkspaceMembership, and a full CRUD API for user-created multi-tenant workspaces.

01

Architecture Decisions

Goal
Introduce CustomWorkspace — user-created workspaces grouping multiple CommCare tenants — by evolving TenantWorkspace rather than introducing a parallel hierarchy.
App rename
Deferred — keep apps/projects
Model design
Evolve TenantWorkspace in-place: OneToOne → M2M tenants, add workspace_type discriminator
Access control
Unified WorkspaceMembership for all workspace types; synced from TenantMembership via signal
Transport
workspace_id in URL path (aligned with parallel URL refactor)
Permission enforcement
DRF WorkspacePermission permission class
Assumption: A parallel refactor adds workspace_id to URL paths (e.g., /api/knowledge/<workspace_id>/). Phase 3 WorkspacePermission slots into that URL structure. Phases 1–2 are independent and can proceed before the URL refactor lands.

WorkspaceTenant NEW
FK  workspace → TenantWorkspace
FK  tenant → Tenant
DT  added_at
unique_together: (workspace, tenant)
Through table for the M2M relationship. Tracks when each tenant was linked to the workspace.
WorkspaceMembership NEW
UUID id
FK  workspace → TenantWorkspace
FK  user → User
Char role (owner|editor|viewer)
FK  invited_by → User (null)
DT  joined_at
unique_together: (workspace, user)
Auto-synced from TenantMembership via signal for type='tenant'. Managed explicitly for type='custom'.
WorkspacePermission NEW
extends BasePermission
has_permission(request, view)
1. Reads workspace_id from URL kwargs
2. 404 if workspace not found
3. 403 if no WorkspaceMembership
4. For custom: re-validates TenantMembership
5. Sets request.workspace (avoids 2nd fetch)
DRF permission class. Requires Phase 3 URL refactor (workspace_id in URL path).

New Factory Methods & Callsite Changes

Factory Methods (new)
create_for_tenant(tenant)
Creates workspace_type='tenant', links M2M, sets name from canonical_name
get_or_create_for_tenant(tenant)
Idempotent — returns (workspace, created). Replaces get_or_create(tenant=...)
primary_tenant()
Returns sole Tenant for type='tenant'. Raises ValueError for type='custom'
